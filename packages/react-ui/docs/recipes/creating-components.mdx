### Step 1: To start, create a new React component

<div className="example code-first preview-margin">

```jsx
import React from 'react'

const Label = props => {
  return (
    <span {...props}>{props.children}</span>
  )
}

render(<Label>Private</Label>) // only for example
```

<ThemeProvider>
  <span>Private</span>
</ThemeProvider>

</div>

Note that we spread props (`...props`) down to the element to allow the user of our component to be able to add any additional properties like `aria-label`, `data-test-id` etc.

Components inside `react-ui` use the `Element` primitive as their base. This gives access to a few utilities to the component.

<div className="example code-first preview-margin">

```jsx
import React from 'react'
import { Element } from 'react-ui'

const Label = props => {
  return (
    <Element {...props}>{props.children}</Element>
  )
}

render(<Label>Private</Label>) // only for example
```

<ThemeProvider>
  <div>Private</div>
</ThemeProvider>

</div>

The first one of these is the `as` prop. The `as` defines what HTML element will be rendered.

<div className="example code-first preview-margin">

```jsx
import React from 'react'
import { Element } from 'react-ui'

const Label = props => {
  return (
    <Element as="span" {...props}>{props.children}</Element>
  )
}

render(<Label>Private</Label>) // only for example
```

<ThemeProvider>
  <span>Private</span>
</ThemeProvider>

</div>

[Read more about the `as` prop](/docs/utilities/as)

### Step 2: Next up is the `css` prop:

You can use the `css` prop to add styles to your component. The styles you add are derived from the theme file provided in your [ThemeProvider](/docs/components/theme-provider).

Let's start with the text color and border of the component.

<div className="example code-first preview-margin">

```jsx
import React from 'react'
import { Element } from 'react-ui'

const styles = {
  color: 'grays.800',
  border: '1px solid',
  borderColor: 'grays.500'
}

const Label = props => {
  return (
    <Element as="span" css={styles} {...props}>{props.children}</Element>
  )
}

render(<Label>Private</Label>) // only for example
```

<ThemeProvider>
  <Element as="span" css={{
    color: 'grays.800',
    border: '1px solid',
    borderColor: 'grays.500'
  }} >
    Private
  </Element>
</ThemeProvider>

</div>

There are a few things to notice here:

1. We are passing an object of _styles tokens_ to our component.

  By using the `Element` primitive, we are able to provide references to the theme tokens, like `grays.800`. `Element` will replace that with the corresponding value from theme and attach a class with those styles inserted.

2. `react-ui` is unopinionated about where you keep this `styles` object.
    
    You can keep it in the same file or a different file. This is an aesthetic choice about separation of concerns and does not affect how you compose components. <br/><br/><small>(If you're curious, the `react-ui` codebase itself, follows a convention of putting styles in a different file with a `.styles.js` suffix, example: `label.styles.js`)</small>

3. This project uses object styles instead of string for styling components.

  Object notation:
  
  ```js
  const styles = {
    color: 'grays.800',
    borderColor: 'grays.500'
  }
  ```

  over a string notation:

  ```css
  const styles = `
    color: grays.800;
    border-color: grays.500;
  `
  ```

  Long term, this project aims to allow allow multiple methods, but for now, focusing on object notation would get us a clean API and stable components faster. [Read more in goals and scope](/docs/goals).

<br/>

Writing out all the styles required for the label, we land up on this. You'll notice a lot of values are taken from the theme smartly.

Too see the whole list of inferred properties, see styled-system's [Reference table](https://styled-system.com/theme-specification#key-reference) which the `Element` primitive utilises.

### Step 3: (optional) Make the component customisable 

If you'd like to utilise the customisation method mentioned in [Customising built-in components](/docs/usage/#customising-built-in-components), there are 2 steps involved:

1. Customisation via `components` in `ThemeProvider`:

    Add a `component` prop to the `Element`. This name should be unique across the application, so that it can be uniquely identified.

    ```jsx
    import React from 'react'
    import { Element } from 'react-ui'
    import styles from './label.styles.js'

    const Label = props => {
      return (
        <Element as="span" component="Label" css={styles} {...props}>{props.children}</Element>
      )
    }
    render(<Label>Private</Label>) // only for example
    ```
    
    <br/>

2. Customisation via `css` prop on the component:

    To allow the usage of `css` prop at the site of usage, we need to make sure it doesn't override the `css` prop inside our component. 
    
    For that, destructure `css` in props and use the `merge` utility from `react-ui` that will _deeply merge_ the two style objects.

    <div className="example code-first preview-margin">
    
    ```jsx
    import React from 'react'
    import { Element, merge } from 'react-ui'
    import styles from './label.styles.js'

    // destructure css out of props
    const Label = ({css, ...props}) => {
      return (
        <Element 
          as="span" 
          css={merge(styles, css)} // deeply merges styles
          {...props}
        >
          {props.children}
        </Element>
      )
    }

    // only for example
    render(
      <>
        <Label marginRight={2}>Private</Label>
        <Label css={{color: 'reds.500'}}>Deleted</Label>
      </>
    )
    ```

    <ThemeProvider>
      <div>
        <Element as="span" marginRight={2} css={{
          display: 'inline-block',
          textTransform: 'uppercase',
          border: '1px solid',
          borderColor: 'grays.500', 
          color: 'grays.800', 
          fontSize: 1, 
          fontWeight: 'semibold', 
          paddingX: 2, 
          paddingY: 1, 
          borderRadius: 2 
        }} >
          Private
        </Element>
        <Element as="span" css={{
          display: 'inline-block',
          textTransform: 'uppercase',
          border: '1px solid',
          borderColor: 'grays.500',
          fontSize: 1, 
          fontWeight: 'semibold', 
          paddingX: 2, 
          paddingY: 1, 
          borderRadius: 2,

### Creating a composite component

More often than creating new building blocks, you'll find yourself creating new components composed of built-in components from `react-ui`.

As an example, let's build a `Breadcrumbs` component.

(This guide assumes you have read the first part about [creating new building blocks](docs/recipes/creating-components/#creating-a-new-building-block))

<br/>

This is what we would like our breadcrumbs component to look like:

<div className="example preview-first preview-margin">

<ThemeProvider>
  <Element
    as="div"
    aria-label="breadcrumb"
    css={{
      fontSize: 3,
      Link: {
        ':after': {
          content: "'/'",
          color: 'grays.700',
          paddingX: 2
        }
      }
    }}
  >
    <Link href="/home">Home</Link>
    <Link href="/settings">Settings</Link>
    <span aria-current>Update profile details</span>
  </Element>
</ThemeProvider>

```jsx
<Breadcrumbs>
  <Link href="/home">Home</Link>
  <Link href="/settings">Settings</Link>
  <span aria-current>Update profile details</span>
</Breadcrumbs>
```

</div>

Our component needs to set the font size of it's component and add a separator(`/`) after each link.

<br/>

**Step 1: Breadcrumb styles**

To begin, let's create a new component like we did earlier on this page with the `Element` primitive.

We have added the font size we want our breadcrumb to have.

```jsx
import React from 'react'
import { Element } from 'react-ui'

const styles = {
  fontSize: 3
}

const Breadcrumbs = props => (
  <Element as="div" css={styles} {...props}>
    {props.children}
  </Element>
)
```

Note: this guide does not show you how to create an accessible breadcrumb. To see the full implementation, see the [Breadcrumbs component](/docs/components/breadcrumbs) from `react-ui`

<br/>

**Step 2: Add additional styles to Link**

The styles we provide to a component build with the `Element` primitive supports nesting.

We can customise what styles are added to the `Link` component when it is present inside the `Breadcrumbs` component like this:

```jsx
import React from 'react'
import { Element } from 'react-ui'

const styles = {
  fontSize: 3,
  // we can target the Link component by its data-component
  Link: { 
    ':after': {
      content: "'/'",
      color: 'grays.700',
      paddingX: 2
    }
  }
}

const Breadcrumbs = props => (
  <Element as="div" css={styles} {...props}>
    {props.children}
  </Element>
)
```

If you'd like to use this method for components you build yourself, refer to the documentation for [making components customisable](/docs/recipes/creating-components/#step-3-optional-make-the-component-customisable)
